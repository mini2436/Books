# Private Reader 接口文档

## 1. 说明

当前后端接口以 REST 为主，统一前缀为：

- `http://localhost:8080/api`

鉴权方式：

- 登录后拿到 `accessToken`
- 之后在请求头中传：

```http
Authorization: Bearer <accessToken>
```

返回格式：

- 成功：JSON
- 错误：默认返回 `{"error":"..."}` 或校验错误对象

## 2. 认证接口

### 2.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要登录：否

请求体：

```json
{
  "username": "admin",
  "password": "admin12345"
}
```

响应示例：

```json
{
  "accessToken": "....",
  "refreshToken": "....",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "SUPER_ADMIN"
  }
}
```

### 2.2 刷新令牌

- 方法：`POST`
- 路径：`/api/auth/refresh`
- 是否需要登录：否

请求体：

```json
{
  "refreshToken": "...."
}
```

### 2.3 退出登录

- 方法：`POST`
- 路径：`/api/auth/logout`
- 是否需要登录：是

说明：

- 会撤销当前 `Authorization` 对应的 access token

## 3. 阅读端接口

### 3.1 获取我的书架

- 方法：`GET`
- 路径：`/api/me/books`
- 是否需要登录：是

返回字段：

- `id`
- `title`
- `author`
- `description`
- `pluginId`
- `format`
- `sourceType`
- `granted`
- `sourceMissing`
- `updatedAt`

### 3.2 获取单本书详情

- 方法：`GET`
- 路径：`/api/me/books/{bookId}`
- 是否需要登录：是

说明：

- 只有被授权的用户才能访问

### 3.3 获取阅读 manifest

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content-manifest`
- 是否需要登录：是

说明：

- 返回插件生成的阅读导航结构
- EPUB / PDF / TXT 会根据插件返回各自格式的结构

### 3.4 获取原始文件

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/file`
- 是否需要登录：是

说明：

- 当前直接返回服务器上的原始文件流
- 会根据格式返回 `epub/pdf/txt` 对应的 `Content-Type`

## 4. 同步接口

## 4.1 推送离线变更

- 方法：`POST`
- 路径：`/api/me/sync/push`
- 是否需要登录：是

请求体由三类数据组成：

- `annotations`
- `bookmarks`
- `progresses`

示例：

```json
{
  "annotations": [
    {
      "clientTempId": "anno-local-1",
      "bookId": 12,
      "action": "CREATE",
      "quoteText": "example quote",
      "noteText": "my note",
      "color": "yellow",
      "anchor": "{\"type\":\"epub-cfi\",\"value\":\"epubcfi(/6/2...)\"}",
      "updatedAt": "2026-04-18T12:00:00Z"
    }
  ],
  "bookmarks": [],
  "progresses": []
}
```

响应示例：

```json
{
  "annotationMappings": {
    "anno-local-1": 101
  },
  "conflicts": []
}
```

冲突说明：

- 如果同一条批注的 `baseVersion` 落后于服务端版本
- 服务端不会直接覆盖，而是返回 `conflicts`
- 客户端需要提示用户选择保留本地、保留服务端、或另存为新批注

### 4.2 拉取增量同步数据

- 方法：`GET`
- 路径：`/api/me/sync/pull`
- 是否需要登录：是

查询参数：

- `cursor`：上次同步完成时拿到的时间游标，单位毫秒

响应字段：

- `cursor`
- `annotations`
- `bookmarks`
- `progresses`

### 4.3 获取某本书的批注

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/annotations`
- 是否需要登录：是

### 4.4 获取某本书的书签

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/bookmarks`
- 是否需要登录：是

### 4.5 更新阅读进度

- 方法：`PUT`
- 路径：`/api/me/books/{bookId}/progress`
- 是否需要登录：是

请求体示例：

```json
{
  "bookId": 12,
  "location": "epubcfi(/6/8!/4/2/14)",
  "progressPercent": 42.5,
  "updatedAt": "2026-04-18T12:10:00Z"
}
```

## 5. 管理端接口

## 5.1 上传书籍

- 方法：`POST`
- 路径：`/api/admin/books/upload`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`
- 请求类型：`multipart/form-data`

字段：

- `file`: 书籍文件

说明：

- 上传后会按插件识别格式
- 新导入的书默认授权给上传者本人

### 5.2 给用户授权某本书

- 方法：`POST`
- 路径：`/api/admin/books/{bookId}/grants`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`

请求体：

```json
{
  "userId": 2
}
```

### 5.3 查看导入任务

- 方法：`GET`
- 路径：`/api/admin/books/import-jobs`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`

## 6. 书库扫描源接口

### 6.1 获取扫描源列表

- 方法：`GET`
- 路径：`/api/admin/library-sources`
- 是否需要登录：是

### 6.2 创建扫描源

- 方法：`POST`
- 路径：`/api/admin/library-sources`
- 是否需要登录：是

请求体：

```json
{
  "name": "NAS Books",
  "rootPath": "Z:\\ebooks",
  "enabled": true
}
```

### 6.3 手动触发扫描

- 方法：`POST`
- 路径：`/api/admin/library-sources/{sourceId}/rescan`
- 是否需要登录：是

说明：

- 会遍历目录中的文件
- 发现新文件时导入
- 已消失的文件会被标记为 `source_missing`

## 7. 用户管理接口

### 7.1 获取用户列表

- 方法：`GET`
- 路径：`/api/admin/users`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN`

### 7.2 创建用户

- 方法：`POST`
- 路径：`/api/admin/users`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN`

请求体：

```json
{
  "username": "reader01",
  "password": "change-me",
  "role": "READER"
}
```

支持角色：

- `SUPER_ADMIN`
- `LIBRARIAN`
- `READER`

### 7.3 更新用户

- 方法：`PATCH`
- 路径：`/api/admin/users/{userId}`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN`

请求体示例：

```json
{
  "enabled": true,
  "role": "LIBRARIAN"
}
```

## 8. 插件接口

### 8.1 获取插件列表

- 方法：`GET`
- 路径：`/api/admin/plugins`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`

返回内容：

- `pluginId`
- `displayName`
- `supportedExtensions`
- `capabilities`

当前内置插件：

- `plugin-epub`
- `plugin-pdf`
- `plugin-txt`

## 9. 权限规则

权限判定规则：

- 认证接口匿名可访问
- `/api/admin/**` 仅管理员或馆员可访问
- `/api/me/**` 需要登录
- 书籍内容、manifest、文件下载都要求用户已拥有该书授权

## 10. 当前已知限制

- Web 前端壳还没有完整接入登录流程
- 管理端页面目前只展示最基础数据，不是完整后台
- 插件的元数据提取还是骨架实现，真实格式解析后续还要增强
- 接口文档当前是手工维护版，后续可以补成 OpenAPI/Swagger
