# Private Reader 运行文档

## 1. 项目组成

Private Reader 当前由 4 个部分组成：

- `backend/`：Kotlin + Spring Boot 3 后端
- `web/`：Next.js 网页阅读端与管理端
- `mobile/`：Capacitor 移动端外壳
- `infra/`：数据库脚本、中间件辅助脚本、运行说明

## 3. 最推荐的本地启动方式

### 3.1 启动中间件

在项目根目录运行：

```powershell
.\infra\scripts\start-infra.ps1
```

或者直接运行：

```powershell
docker compose up -d postgres redis rabbitmq
```

默认端口：

- PostgreSQL: `5432`
- Redis: `6379`
- RabbitMQ AMQP: `5672`
- RabbitMQ 管理界面: `15672`

### 3.2 启动后端

建议先设置当前终端的 JDK 21：

```powershell
$env:JAVA_HOME='C:\Users\mini2436\Tools\temurin21\jdk-21.0.10+7'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
```

进入后端目录并启动：

```powershell
cd .\backend
.\gradlew bootRun
```

默认后端地址：

- API: `http://localhost:8080`
- 健康检查: `http://localhost:8080/actuator/health`

### 3.3 启动前端开发模式

新开一个终端，进入前端目录：

```powershell
cd .\web
npm install
npm run dev
```

默认前端地址：

- Web: `http://localhost:3000`

如果新终端里 `npm` 还不可用，可以临时执行：

```powershell
$env:PATH='C:\Users\mini2436\Tools\node-v20.19.5-win-x64;' + $env:PATH
```

### 3.4 构建前端静态产物

项目已经提供了脚本：


构建后产物目录为：

## 9. 故障排查

### 9.1 `npm` 命令找不到

原因：

- 新增 PATH 后当前终端没有刷新

处理：

- 重新打开一个终端
- 或者临时执行：

```powershell
$env:PATH='C:\Users\mini2436\Tools\node-v20.19.5-win-x64;' + $env:PATH
```

### 9.2 后端启动失败，提示数据库连接不上

检查：

- PostgreSQL 容器是否已启动
- `5432` 端口是否可访问
- `SPRING_DATASOURCE_*` 环境变量是否被覆盖

### 9.3 Native 构建失败

检查：

- `GRAALVM_HOME` 是否设置
- 是否使用了 GraalVM 自带的 `native-image`
- Windows 下 `cl.exe` 是否可用

### 9.4 登录成功但前端页面请求 401

原因：

- 当前前端壳还没有完整接入登录态存储与 `Authorization` 头透传

建议：

- 先用接口调试工具验证 API
- 或在下一轮开发中补上 Web 登录流程和 token 注入
