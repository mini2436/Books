# 第一阶段详细功能文档

## 1. 范围说明

第一阶段围绕四条主链路展开：

- 用户登录与会话恢复
- 书架到阅读器的阅读闭环
- 批注、书签、阅读进度同步
- 管理员在 Flutter App 内完成后台管理与资源扫描

## 2. 角色说明

### 普通用户

- 登录系统
- 浏览书架
- 进入阅读器阅读
- 查看目录与切换章节
- 创建和查看批注
- 管理书签
- 保存阅读设置与阅读进度

### 管理员

- 具备普通用户全部能力
- 进入后台管理中心
- 管理书籍、批注与扫描源

### 超级管理员

- 具备管理员全部能力
- 管理用户启停
- 调整用户角色

## 3. Flutter App 功能明细

### 3.1 登录

- 支持账号密码登录。
- 登录成功后保存会话令牌。
- App 重启后尝试恢复会话。

### 3.2 书架

- 拉取当前用户可见书籍列表。
- 展示封面、标题、作者等摘要信息。
- 点击书籍进入阅读器。

### 3.3 阅读器

- 加载图书详情与统一正文内容。
- 显示当前章节标题与阅读内容。
- 支持上一章、下一章切换。
- 支持目录面板。
- 支持手机与平板不同布局。
- 手机端已改为浮层式顶部操作栏与底部进度栏。
- 当前主支持统一正文阅读体验的格式为 `TXT / EPUB`。
- 对未接入统一正文链路的内容，阶段一不作为 App 主阅读目标。

### 3.4 批注与书签

- 阅读中可打开批注面板与书签面板。
- 批注中心支持查看整理已有批注。
- 书签与批注数据走统一同步链路。

### 3.5 阅读设置

- 支持阅读偏好项存储。
- 支持设置面板在阅读器中即时生效。

### 3.6 个人中心

- 展示当前账号信息。
- 提供配置入口与会话相关操作。

## 4. 管理后台功能明细

### 4.1 后台首页与分区

- 后台入口位于 Flutter App 的后台 Tab。
- 当前分区包括用户、角色、书籍、批注、资源扫描。
- 顶部提供摘要统计、刷新、分区切换与消息提示。

### 4.2 用户管理

- 超级管理员可查看用户列表。
- 可切换用户启用状态。
- 可创建新用户。

### 4.3 角色管理

- 超级管理员可查看角色摘要。
- 可调整用户角色。

### 4.4 书籍管理

- 支持图书搜索。
- 支持按分组筛选。
- 支持当前结果集全选。
- 支持批量删除已选图书。
- 支持进入图书详情管理页。
- 支持上传图书。

### 4.5 批注管理

- 管理员可查看系统内批注数据。
- 支持运营和清理相关的管理动作。

### 4.6 资源扫描

- 支持维护扫描源。
- 当前重点场景为 WebDAV 书库接入。
- 支持触发扫描任务并将结果入库。
- 该能力用于替代人工逐本导入的低效率流程。

## 5. 后端能力明细

### 5.1 认证鉴权

- 提供登录接口。
- 提供 Bearer Token 鉴权过滤。
- 提供角色与权限校验。

### 5.2 图书服务

- 提供读者侧图书接口。
- 提供管理侧图书接口。
- 提供封面访问与图书详情相关能力。

### 5.3 同步服务

- 提供批注、书签、阅读进度相关同步接口。
- 支撑 App 离线后回传本地变更。

### 5.4 后台管理服务

- 提供用户与角色管理接口。
- 提供批注管理接口。
- 提供图书管理接口。

### 5.5 扫描与导入

- 提供资源扫描源管理接口。
- 提供扫描执行与入库流程。
- 已纳入后台工作流。

## 6. 非功能特性

### 6.1 终端体验

- Android 手机与平板为当前优先目标。
- 平板阅读器与后台都做了专门适配。

### 6.2 可维护性

- 后端采用多模块拆分。
- Flutter 采用按 `app / data / features / shared` 的分层目录。

### 6.3 交付策略

- 以 Flutter App 为主交付端。
- 原 Web 端保留为迁移参考，不再承接新功能。

## 7. 第一阶段遗留项

- Flutter Web/Desktop 尚未正式接管原 Web 端能力。
- 原 Web 的部分后台和阅读交互仍需迁移映射。
- 多端统一设计 token、输入交互与窗口化布局仍需继续抽象。
