# Private Reader 接口文档

## 1. 说明

当前后端接口以 REST 为主，默认本地地址为：

- `http://localhost:8080`
- 业务接口统一位于 `/api` 下

鉴权方式：

- 登录后获得 `accessToken`
- 需要登录的接口在请求头中传入：

```http
Authorization: Bearer <accessToken>
```

角色说明：

- `SUPER_ADMIN`：超级管理员，可管理用户、角色和全部后台能力
- `LIBRARIAN`：馆员，可管理图书、扫描源、批注和书签
- `READER`：读者，仅访问自己的阅读、同步和授权书籍

统一响应：

- 成功：返回 JSON 或文件流
- 错误：默认返回 `{"error":"..."}` 或 `{"message":"..."}` 等 JSON

## 2. 认证接口

### 2.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要登录：否

请求体：

```json
{
  "username": "admin",
  "password": "admin12345"
}
```

响应示例：

```json
{
  "accessToken": "....",
  "refreshToken": "....",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "SUPER_ADMIN"
  }
}
```

### 2.2 刷新令牌

- 方法：`POST`
- 路径：`/api/auth/refresh`
- 是否需要登录：否

请求体：

```json
{
  "refreshToken": "...."
}
```

说明：

- 使用 `refreshToken` 换取新的 `accessToken` 和 `refreshToken`
- 原刷新令牌会被撤销

### 2.3 退出登录

- 方法：`POST`
- 路径：`/api/auth/logout`
- 是否需要登录：是

说明：

- 会撤销当前 `Authorization` 对应的 access token

响应示例：

```json
{
  "success": true
}
```

## 3. 阅读端接口

### 3.1 获取我的书架

- 方法：`GET`
- 路径：`/api/me/books`
- 是否需要登录：是

说明：

- `SUPER_ADMIN` 和 `LIBRARIAN` 默认可访问全部书籍
- `READER` 仅返回被显式授权的书籍

返回字段：

- `id`
- `title`
- `author`
- `description`
- `pluginId`
- `format`
- `sourceType`
- `granted`
- `sourceMissing`
- `updatedAt`

### 3.2 获取单本书详情

- 方法：`GET`
- 路径：`/api/me/books/{bookId}`
- 是否需要登录：是

说明：

- 只有拥有该书访问权限的用户才能访问
- 详情接口会返回移动端统一正文阅读所需字段

响应示例：

```json
{
  "id": 12,
  "title": "示例图书",
  "author": "作者",
  "description": "简介",
  "pluginId": "plugin-epub",
  "format": "epub",
  "sourceType": "MANAGED_UPLOAD",
  "manifest": {
    "format": "epub",
    "toc": [
      { "title": "第一章", "href": "chapter-0" }
    ],
    "primaryLocation": "chapter-0"
  },
  "capabilities": ["READ_ONLINE"],
  "sourceMissing": false,
  "hasStructuredContent": true,
  "contentModel": "UNIFIED_V1",
  "latestContentVersionId": 31
}
```

### 3.3 获取阅读 manifest

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content-manifest`
- 是否需要登录：是

说明：

- 返回插件生成的阅读导航结构
- Web 旧阅读链路仍会消费该接口
- 如果插件没有生成 manifest，可能返回 `null`

### 3.4 获取统一正文目录

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content`
- 是否需要登录：是

用途：

- 判断该书是否存在统一正文
- 获取章节摘要列表，供移动端做目录与章节懒加载

响应示例：

```json
{
  "bookId": 12,
  "contentModel": "UNIFIED_V1",
  "contentVersionId": 31,
  "hasStructuredContent": true,
  "chapters": [
    {
      "chapterIndex": 0,
      "title": "第一章",
      "anchor": "chapter-0"
    }
  ]
}
```

### 3.5 获取统一正文单章

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content/chapters/{chapterIndex}`
- 是否需要登录：是

用途：

- 章节懒加载
- 目录定位后按需加载正文块

响应示例：

```json
{
  "bookId": 12,
  "contentModel": "UNIFIED_V1",
  "contentVersionId": 31,
  "hasStructuredContent": true,
  "chapterIndex": 0,
  "title": "第一章",
  "anchor": "chapter-0",
  "blocks": [
    {
      "blockIndex": 0,
      "type": "paragraph",
      "anchor": "chapter-0-block-1",
      "text": "正文内容",
      "plainText": "正文内容",
      "meta": {}
    }
  ]
}
```

说明：

- 当前块类型主要包括：`heading`、`paragraph`、`quote`、`divider`
- 移动端首期统一正文阅读器主要面向 `TXT / EPUB`

### 3.6 获取原始文件

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/file`
- 是否需要登录：是

说明：

- 返回服务器上的原始文件流
- 用于 Web 旧阅读链路和 PDF 回退链路

### 3.7 获取封面

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/cover`
- 是否需要登录：是

说明：

- 有封面时返回图片流
- 无封面时返回 `204`

### 3.8 获取正文资源

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content/resources/{resourceId}`
- 是否需要登录：是

说明：

- 返回结构化正文或 manifest 中引用的二进制资源
- 当前主要用于图片等内嵌资源
- 找不到资源时返回 `404`

## 4. 同步接口

### 4.1 推送离线变更

- 方法：`POST`
- 路径：`/api/me/sync/push`
- 是否需要登录：是

请求体字段：

- `annotations`：批注变更
- `bookmarks`：书签变更
- `progresses`：阅读进度变更

批注 action：

- `CREATE`
- `UPDATE`
- `DELETE`

示例：

```json
{
  "annotations": [
    {
      "clientTempId": "anno-local-1",
      "bookId": 12,
      "action": "CREATE",
      "quoteText": "example quote",
      "noteText": "my note",
      "color": "#C3924A",
      "anchor": "chapter-0-block-1",
      "updatedAt": "2026-04-18T12:00:00Z"
    }
  ],
  "bookmarks": [],
  "progresses": []
}
```

响应字段：

- `annotationMappings`：客户端临时批注 ID 到服务端 ID 的映射
- `conflicts`：版本冲突列表

说明：

- 新移动端默认使用统一正文 anchor
- 批注更新会校验 `baseVersion`，版本不一致会返回冲突

### 4.2 拉取增量同步数据

- 方法：`GET`
- 路径：`/api/me/sync/pull`
- 是否需要登录：是

查询参数：

- `cursor`：上次同步完成时拿到的时间游标，单位毫秒，可选

响应字段：

- `cursor`
- `annotations`
- `bookmarks`
- `progresses`

### 4.3 获取某本书的批注

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/annotations`
- 是否需要登录：是

说明：

- 返回当前用户在该书上的批注列表

### 4.4 获取某本书的书签

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/bookmarks`
- 是否需要登录：是

说明：

- 返回当前用户在该书上的书签列表

### 4.5 更新阅读进度

- 方法：`PUT`
- 路径：`/api/me/books/{bookId}/progress`
- 是否需要登录：是

请求体示例：

```json
{
  "bookId": 12,
  "location": "chapter-0",
  "progressPercent": 42.5,
  "updatedAt": "2026-04-18T12:10:00Z"
}
```

说明：

- 路径中的 `bookId` 必须和请求体中的 `bookId` 一致

## 5. 后台书籍管理接口

以下接口均位于 `/api/admin/books` 下，需要登录，角色要求为 `SUPER_ADMIN` 或 `LIBRARIAN`。

### 5.1 获取后台书籍列表

- 方法：`GET`
- 路径：`/api/admin/books`

### 5.2 获取后台书籍详情

- 方法：`GET`
- 路径：`/api/admin/books/{bookId}`

### 5.3 上传书籍

- 方法：`POST`
- 路径：`/api/admin/books/upload`
- 请求类型：`multipart/form-data`
- 表单字段：`file`

说明：

- 上传后会根据文件格式插件导入书籍
- 上传人会自动获得该书访问权限

### 5.4 更新书籍

- 方法：`PATCH`
- 路径：`/api/admin/books/{bookId}`

请求体：

```json
{
  "groupName": "经典文学"
}
```

说明：

- 当前用于维护后台书籍分组
- `groupName` 传 `null` 或空字符串会清空分组

### 5.5 重建结构化正文

- 方法：`POST`
- 路径：`/api/admin/books/{bookId}/content/rebuild`

说明：

- 重新读取书籍源文件并抽取统一正文

### 5.6 批量删除书籍

- 方法：`POST`
- 路径：`/api/admin/books/bulk-delete`

请求体：

```json
{
  "bookIds": [12, 13]
}
```

响应示例：

```json
{
  "success": true,
  "deletedCount": 2
}
```

### 5.7 给用户授权某本书

- 方法：`POST`
- 路径：`/api/admin/books/{bookId}/grants`

请求体：

```json
{
  "userId": 3
}
```

### 5.8 查看某本书可见用户

- 方法：`GET`
- 路径：`/api/admin/books/{bookId}/viewers`

返回字段：

- `userId`
- `username`
- `role`
- `enabled`
- `accessSource`：`GLOBAL_ROLE` 或 `EXPLICIT_GRANT`
- `grantedAt`

### 5.9 解除用户书籍授权

- 方法：`DELETE`
- 路径：`/api/admin/books/{bookId}/grants/{userId}`

说明：

- 只移除显式授权
- 全局角色用户仍会因为角色权限可见

### 5.10 获取可授权用户

- 方法：`GET`
- 路径：`/api/admin/books/grantable-users`

说明：

- 返回启用状态的用户列表

### 5.11 查看导入任务

- 方法：`GET`
- 路径：`/api/admin/books/import-jobs`

说明：

- 返回最近 100 条导入任务

## 7. 书库扫描源接口

以下接口均位于 `/api/admin/library-sources` 下，需要登录，角色要求为 `SUPER_ADMIN` 或 `LIBRARIAN`。

### 7.1 获取扫描源列表

- 方法：`GET`
- 路径：`/api/admin/library-sources`

### 7.2 创建扫描源

- 方法：`POST`
- 路径：`/api/admin/library-sources`

本地文件夹请求体：

```json
{
  "name": "本地书库",
  "sourceType": "WATCHED_FOLDER",
  "rootPath": "D:/Books",
  "enabled": true,
  "scanIntervalMinutes": 60
}
```

WebDAV 请求体：

```json
{
  "name": "NAS 书库",
  "sourceType": "WEBDAV",
  "baseUrl": "https://example.com/dav/",
  "remotePath": "/books",
  "username": "reader",
  "password": "secret",
  "enabled": true,
  "scanIntervalMinutes": 60
}
```

### 7.3 更新扫描源

- 方法：`PATCH`
- 路径：`/api/admin/library-sources/{sourceId}`

说明：

- 请求体字段和创建扫描源一致

### 7.4 手动触发扫描

- 方法：`POST`
- 路径：`/api/admin/library-sources/{sourceId}/rescan`

响应示例：

```json
{
  "sourceId": 1,
  "imported": 3,
  "missingMarked": 1
}
```

- `role` 和 `enabled` 均为可选字段

## 9. 插件接口

### 9.1 获取插件列表

- 方法：`GET`
- 路径：`/api/admin/plugins`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`

返回字段：

- `pluginId`
- `displayName`
- `supportedExtensions`
- `capabilities`

当前内置插件：

- `plugin-epub`
- `plugin-pdf`
- `plugin-txt`

## 10. 权限规则

- `/api/auth/login`、`/api/auth/refresh` 匿名可访问
- `/api/auth/logout` 需要登录
- `/api/me/**` 需要登录
- `/api/admin/users/**` 仅 `SUPER_ADMIN` 可访问
- `/api/admin/**` 仅 `SUPER_ADMIN` 或 `LIBRARIAN` 可访问
- 书籍详情、manifest、统一正文、文件、封面和内容资源都要求用户拥有该书访问权限

## 11. 当前已知限制

- 移动端首期主要支持 `TXT / EPUB` 统一正文阅读
- `PDF` 当前仍保留桌面 Web 阅读路径
- 统一正文当前不覆盖复杂富文本、图片块和 PDF 版面阅读
- 接口文档当前仍是手工维护版，后续可补 OpenAPI / Swagger
