# Private Reader 运行文档

## 1. 项目组成

Private Reader 当前由 3 个部分组成：

- `backend/`：Kotlin + Spring Boot 3 后端
- `mobile/`：Flutter 客户端，Android 手机/平板优先，并承接后续 Web/Desktop 多端开发
- `infra/`：数据库脚本、中间件辅助脚本、运行说明

旧 Next.js Web 端已经从仓库中移除；后续浏览器端和桌面端能力统一在 `mobile/` Flutter 工程中演进。

当前项目默认使用这些本机工具：

- JDK 21：`C:\Users\mini2436\Tools\temurin21\jdk-21.0.10+7`
- GraalVM：`C:\Users\mini2436\Tools\graalvm\graalvm-community-openjdk-25.0.2+10.1`

后端依赖以下中间件：

- Redis 7

本项目已经在 [docker-compose.yml](../docker-compose.yml) 中定义好了这些服务，开发环境不需要单独安装。

### 2.3 代理说明

如果 Flutter Pub 或 Android Gradle 依赖下载失败，可临时使用本机代理：

- `http://127.0.0.1:10808`

推荐只在当前终端设置，不写进仓库：

```powershell
$env:HTTP_PROXY='http://127.0.0.1:10808'
$env:HTTPS_PROXY='http://127.0.0.1:10808'
$env:GRADLE_OPTS='-Dhttp.proxyHost=127.0.0.1 -Dhttp.proxyPort=10808 -Dhttps.proxyHost=127.0.0.1 -Dhttps.proxyPort=10808'
```

## 3. 最推荐的本地启动方式

### 3.1 启动中间件

在项目根目录运行：

```powershell
.\infra\scripts\start-infra.ps1
```

或者直接运行：

```powershell
docker compose up -d postgres redis rabbitmq
```

默认端口：

- PostgreSQL: `5432`
- Redis: `6379`
- RabbitMQ AMQP: `5672`
- RabbitMQ 管理界面: `15672`

### 3.2 启动后端

建议先设置当前终端的 JDK 21：

```powershell
$env:JAVA_HOME='C:\Users\mini2436\Tools\temurin21\jdk-21.0.10+7'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
```

进入后端目录并启动：

```powershell
cd .\backend
.\gradlew bootRun
```

默认后端地址：

- API: `http://localhost:8080`
- 健康检查: `http://localhost:8080/actuator/health`

### 3.3 启动 Flutter APP

新开一个终端，进入移动端目录：

```powershell
$env:JAVA_HOME='C:\Users\mini2436\Tools\temurin21\jdk-21.0.10+7'
$env:PATH='C:\Users\mini2436\Tools\flutter-sdk\flutter\bin;' + $env:JAVA_HOME + '\bin;' + $env:PATH

cd .\mobile
flutter pub get
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8080
```

说明：

- Android 模拟器访问宿主机后端时，默认使用 `http://10.0.2.2:8080`
- Android 真机调试时，请把 `API_BASE_URL` 改成宿主机局域网 IP，例如 `http://192.168.1.10:8080`
- Flutter 端支持 `TXT / EPUB` 的统一正文阅读器
- `PDF` 或未生成统一正文的书会在 APP 内走对应的 Flutter 阅读能力或提示当前格式暂不可用

### 3.4 启动 Flutter Windows 桌面端

新开一个终端，进入移动端目录：

```powershell
$env:PATH='C:\Users\mini2436\Tools\flutter-sdk\flutter\bin;' + $env:PATH

cd .\mobile
flutter pub get
flutter run -d windows
```

说明：

- Windows 桌面端访问本机后端时使用 `http://localhost:8080`
- 如需覆盖默认服务地址，可继续传入 `--dart-define=API_BASE_URL=http://localhost:8080`
- Windows v1 先使用 Flutter 原生正文渲染路径，Android 仍保留 WebView 阅读器路径

## 9. 故障排查

### 9.1 `flutter` 命令找不到

```powershell
$env:PATH='C:\Users\mini2436\Tools\flutter-sdk\flutter\bin;' + $env:PATH
```

### 9.2 Flutter / Gradle 下载依赖失败

- 检查代理 `http://127.0.0.1:10808` 是否可用
- 仅在当前终端设置 `HTTP_PROXY` / `HTTPS_PROXY` / `GRADLE_OPTS`
- 关闭代理后重新开一个终端，避免影响 `flutter test` 等本地进程通信

### 9.3 Android 模拟器无法访问后端

- 确认后端在宿主机 `8080` 端口正常启动
- 模拟器请使用 `http://10.0.2.2:8080`
- 真机请使用宿主机局域网 IP，而不是 `localhost`

### 9.4 后端启动失败，提示数据库连接不上

- 检查 PostgreSQL 容器是否已启动
- 检查 `5432` 端口是否可访问
- 检查 `SPRING_DATASOURCE_*` 环境变量是否被覆盖
