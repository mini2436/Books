# Private Reader 接口文档

## 1. 说明

当前后端接口以 REST 为主，统一前缀为：

- `http://localhost:8080/api`

鉴权方式：

- 登录后拿到 `accessToken`
- 之后在请求头中传：

```http
Authorization: Bearer <accessToken>
```

返回格式：

- 成功：JSON
- 错误：默认返回 `{"error":"..."}` 或 `{"message":"..."}` 等 JSON

## 2. 认证接口

### 2.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要登录：否

请求体：

```json
{
  "username": "admin",
  "password": "admin12345"
}
```

响应示例：

```json
{
  "accessToken": "....",
  "refreshToken": "....",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "SUPER_ADMIN"
  }
}
```

### 2.2 刷新令牌

- 方法：`POST`
- 路径：`/api/auth/refresh`
- 是否需要登录：否

请求体：

```json
{
  "refreshToken": "...."
}
```

### 2.3 退出登录

- 方法：`POST`
- 路径：`/api/auth/logout`
- 是否需要登录：是

说明：

- 会撤销当前 `Authorization` 对应的 access token

## 3. 阅读端接口

### 3.1 获取我的书架

- 方法：`GET`
- 路径：`/api/me/books`
- 是否需要登录：是

返回字段：

- `id`
- `title`
- `author`
- `description`
- `pluginId`
- `format`
- `sourceType`
- `granted`
- `sourceMissing`
- `updatedAt`

### 3.2 获取单本书详情

- 方法：`GET`
- 路径：`/api/me/books/{bookId}`
- 是否需要登录：是

说明：

- 只有被授权的用户才能访问
- 详情接口会同时返回移动端统一正文分流所需字段

新增或关键字段：

- `hasStructuredContent`
- `contentModel`
- `latestContentVersionId`
- `manifest`
- `capabilities`

响应示例：

```json
{
  "id": 12,
  "title": "示例图书",
  "author": "作者",
  "description": "简介",
  "pluginId": "plugin-epub",
  "format": "epub",
  "sourceType": "MANAGED_UPLOAD",
  "manifest": {
    "format": "epub",
    "toc": [
      { "title": "第一章", "href": "chapter-0" }
    ],
    "primaryLocation": "chapter-0"
  },
  "capabilities": ["READ_ONLINE"],
  "sourceMissing": false,
  "hasStructuredContent": true,
  "contentModel": "UNIFIED_V1",
  "latestContentVersionId": 31
}
```

### 3.3 获取阅读 manifest

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content-manifest`
- 是否需要登录：是

说明：

- 返回插件生成的阅读导航结构
- Web 旧阅读链路仍会消费该接口

### 3.4 获取统一正文目录

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content`
- 是否需要登录：是

用途：

- 判断该书是否存在统一正文
- 获取章节摘要列表，供移动端做目录与章节懒加载

响应示例：

```json
{
  "bookId": 12,
  "contentModel": "UNIFIED_V1",
  "contentVersionId": 31,
  "hasStructuredContent": true,
  "chapters": [
    {
      "chapterIndex": 0,
      "title": "第一章",
      "anchor": "chapter-0"
    }
  ]
}
```

### 3.5 获取统一正文单章

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/content/chapters/{chapterIndex}`
- 是否需要登录：是

用途：

- 章节懒加载
- 目录定位后按需加载正文块

响应示例：

```json
{
  "bookId": 12,
  "contentModel": "UNIFIED_V1",
  "contentVersionId": 31,
  "hasStructuredContent": true,
  "chapterIndex": 0,
  "title": "第一章",
  "anchor": "chapter-0",
  "blocks": [
    {
      "blockIndex": 0,
      "type": "paragraph",
      "anchor": "chapter-0-block-1",
      "text": "正文内容",
      "plainText": "正文内容",
      "meta": {}
    }
  ]
}
```

说明：

- 当前块类型主要包括：`heading`、`paragraph`、`quote`、`divider`
- 移动端首期统一正文阅读器仅支持 `TXT / EPUB`

### 3.6 获取原始文件

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/file`
- 是否需要登录：是

说明：

- 当前直接返回服务器上的原始文件流
- 仍作为 Web 旧阅读链路和 PDF 回退链路的基础接口

### 3.7 获取封面

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/cover`
- 是否需要登录：是

说明：

- 有封面时返回图片流
- 无封面时可能返回 `204`

## 4. 同步接口

### 4.1 推送离线变更

- 方法：`POST`
- 路径：`/api/me/sync/push`
- 是否需要登录：是

请求体由三类数据组成：

- `annotations`
- `bookmarks`
- `progresses`

示例：

```json
{
  "annotations": [
    {
      "clientTempId": "anno-local-1",
      "bookId": 12,
      "action": "CREATE",
      "quoteText": "example quote",
      "noteText": "my note",
      "color": "#C3924A",
      "anchor": "chapter-0-block-1",
      "updatedAt": "2026-04-18T12:00:00Z"
    }
  ],
  "bookmarks": [],
  "progresses": []
}
```

说明：

- 新移动端默认使用统一正文 anchor，不再写入 `epubcfi(...)` 或 `txt-page:*`
- 服务端仍保留兼容旧位置字符串的能力

### 4.2 拉取增量同步数据

- 方法：`GET`
- 路径：`/api/me/sync/pull`
- 是否需要登录：是

查询参数：

- `cursor`：上次同步完成时拿到的时间游标，单位毫秒

响应字段：

- `cursor`
- `annotations`
- `bookmarks`
- `progresses`

### 4.3 获取某本书的批注

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/annotations`
- 是否需要登录：是

### 4.4 获取某本书的书签

- 方法：`GET`
- 路径：`/api/me/books/{bookId}/bookmarks`
- 是否需要登录：是

### 4.5 更新阅读进度

- 方法：`PUT`
- 路径：`/api/me/books/{bookId}/progress`
- 是否需要登录：是

请求体示例：

```json
{
  "bookId": 12,
  "location": "chapter-0",
  "progressPercent": 42.5,
  "updatedAt": "2026-04-18T12:10:00Z"
}
```

## 5. 管理端接口

### 5.1 上传书籍

- 方法：`POST`
- 路径：`/api/admin/books/upload`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`
- 请求类型：`multipart/form-data`

### 5.2 给用户授权某本书

- 方法：`POST`
- 路径：`/api/admin/books/{bookId}/grants`
- 是否需要登录：是
- 角色要求：`SUPER_ADMIN` 或 `LIBRARIAN`

### 5.3 查看导入任务

- 方法：`GET`
- 路径：`/api/admin/books/import-jobs`
- 是否需要登录：是

## 6. 书库扫描源接口

### 6.1 获取扫描源列表

- 方法：`GET`
- 路径：`/api/admin/library-sources`

### 6.2 创建扫描源

- 方法：`POST`
- 路径：`/api/admin/library-sources`

### 6.3 手动触发扫描

- 方法：`POST`
- 路径：`/api/admin/library-sources/{sourceId}/rescan`

## 7. 用户管理接口

### 7.1 获取用户列表

- 方法：`GET`
- 路径：`/api/admin/users`

### 7.2 创建用户

- 方法：`POST`
- 路径：`/api/admin/users`

### 7.3 更新用户

- 方法：`PATCH`
- 路径：`/api/admin/users/{userId}`

## 8. 插件接口

### 8.1 获取插件列表

- 方法：`GET`
- 路径：`/api/admin/plugins`

当前内置插件：

- `plugin-epub`
- `plugin-pdf`
- `plugin-txt`

## 9. 权限规则

- 认证接口匿名可访问
- `/api/admin/**` 仅管理员或馆员可访问
- `/api/me/**` 需要登录
- 书籍内容、manifest、统一正文、文件下载都要求用户已拥有该书授权

## 10. 当前已知限制

- 移动端首期仅支持 `TXT / EPUB` 统一正文阅读
- `PDF` 当前仍保留桌面 Web 阅读路径
- 统一正文当前不覆盖复杂富文本、图片块和 PDF 版面阅读
- 接口文档当前仍是手工维护版，后续可补 OpenAPI / Swagger
