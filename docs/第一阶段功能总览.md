# 第一阶段功能总览

## 文档目标

本文档用于确认 Private Reader 第一阶段已经交付的业务能力、当前边界，以及进入下一阶段前的冻结范围。

## 第一阶段结论

第一阶段以“后端能力跑通 + Flutter App 完成可用闭环 + 管理后台在移动端可落地”为目标，当前已达到可提交、可继续演进的状态。

## 已完成能力

### 1. 账号与权限

- 支持登录、令牌会话恢复与基础鉴权。
- 支持管理员与普通用户角色区分。
- 支持超级管理员管理用户启停与角色调整。

### 2. 书架与阅读

- Flutter App 已具备登录、书架、书籍详情进入阅读的完整路径。
- 阅读器支持统一正文阅读流程。
- 已支持目录、章节切换、书签、批注、阅读设置、阅读进度展示。
- 手机与平板采用统一 Flutter 代码基线，并已针对平板交互做过多轮适配。

### 3. 同步与离线基础

- 已具备批注、书签、阅读进度的同步基础能力。
- App 侧已有离线队列、设置存储、会话存储与同步协调器。

### 4. 管理后台

- Flutter App 侧已支持后台管理入口。
- 已支持用户管理、角色管理、书籍管理、批注管理。
- 已新增“资源扫描”能力，替代原先可见路径中的“书签管理”定位。
- 已支持 WebDAV 扫描源维护、扫描任务发起与导入入库。

### 5. 后端插件化图书处理

- 后端已经按模块拆分 `plugin-api / plugin-epub / plugin-pdf / plugin-txt`。
- 图书格式处理通过编译期集成，不依赖运行时动态插件装载。
- 后端已具备面向原生镜像的 AOT 友好约束。

## 本阶段重点成果

- Flutter 后台管理中心已经可在手机端直接使用。
- WebDAV 资源扫描打通了“扫描源配置 -> 扫描任务 -> 图书入库”的管理链路。
- 阅读器手机端交互已从“硬切换上下栏”优化为“顶部/底部浮层淡入淡出”。
- Tab 壳层切换已改为更安全的 `IndexedStack + Overlay` 方案，降低复杂树动画导致的崩溃风险。

## 本阶段暂不覆盖

- 不继续保留或投入原 Next.js Web 端。
- 不将原 Web 端作为后续主交付面。
- 不在本阶段完成 Flutter Web/Desktop 正式产物。
- 不在本阶段扩展更复杂的后台工作流编排、运营看板或多租户隔离能力。

## 代码基线结论

- 主线业务基线：`backend/` + `mobile/`
- 下一阶段方向：以 Flutter 为统一前端基座，向 Web/Desktop 延展

## 建议的下一阶段目标

- 梳理 Flutter Web/Desktop 的平台差异清单。
- 将现有 App Shell、阅读器、后台中心抽象为更稳定的跨端壳层。
- 在 `mobile/` 工程内逐步补齐 Flutter Web/Desktop 的运行、构建与交互适配。
